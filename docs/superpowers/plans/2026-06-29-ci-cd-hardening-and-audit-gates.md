# CI/CD hardening and audit gates implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the PR quality gate catch formatting drift, dependency advisories, docs build failures, Docker startup failures, and high or critical filesystem security findings.

**Architecture:** Keep `.github/workflows/ci.yml` as the central PR gate. Fix the current failing baseline first, then add audit, docs build, Docker smoke, and Trivy filesystem scanning without changing release publishing or image tag policy.

**Tech Stack:** Bun, oxfmt, oxlint, Astro/Starlight docs, Docker, GitHub Actions, Trivy Action v0.36.0.

---

## Scope

Implement only CI hardening and audit gates.

In scope:

- Fix the existing format failure in `docs/superpowers/plans/2026-06-29-release-management.md`.
- Fix the docs-site `form-data` advisory using a Bun package override.
- Add root and docs-site audit checks to CI.
- Add docs build to the main CI gate.
- Add a Docker build/start smoke check.
- Add Trivy filesystem scanning with SARIF upload.
- Add docs workflow audit coverage.
- Update contributor-facing checklist text to match the new gates.

Out of scope:

- Docker publishing tag changes (`latest`, `edge`, `sha-*`).
- Multi-platform image publishing.
- Helm chart or Helm publishing.
- `/health`, TLS, S3 integration, and real Nx e2e.
- Trivy image scanning. Add image scanning when the Docker publishing workflow is redesigned.

## Current baseline

Plan 1 left the worktree clean, but these checks are currently red:

- `bun run format --check` fails on `docs/superpowers/plans/2026-06-29-release-management.md`.
- `cd docs-site && bun audit` fails on `starlight-openapi -> httpsnippet -> form-data@4.0.4`, advisory `GHSA-hmw2-7cc7-3qxx`.

A temporary copy confirmed that adding `overrides.form-data=4.0.6` in `docs-site/package.json`, then running `bun install`, updates the lockfile and makes `docs-site` audit pass.

## Source checks

- Bun docs: top-level `overrides` can pin metadependencies, and `bun install --frozen-lockfile`/`bun ci` fails when `package.json` and `bun.lock` are out of sync.
- GitHub advisory `GHSA-hmw2-7cc7-3qxx`: `form-data` is affected in `>=4.0.0 <4.0.6`; `4.0.6` is patched.
- Trivy Action docs: filesystem scans use `scan-type: fs` and `scan-ref: .`; SARIF output uses `format: sarif` plus an output file; `exit-code: '1'` fails the job when matching findings exist; `limit-severities-for-sarif: true` keeps SARIF limited to the configured severity list.
- GitHub tag checks: `aquasecurity/trivy-action` tag `v0.36.0` resolves to `a9c7b0f06e461e9d4b4d1711f154ee024b8d7ab8`, `oven-sh/setup-bun` tag `v2.2.0` resolves to `0c5077e51419868618aeaa5fe8019c62421857d6`, and `actions/checkout` tag `v7.0.0` resolves to `9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0`.

## Task 3: Expand the main CI gate

**Files:**

- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Replace `.github/workflows/ci.yml`**

Replace `.github/workflows/ci.yml` with this exact content:

```yaml
name: CI

on:
  push:
    branches: ['main']
  pull_request:

permissions:
  contents: read

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7.0.0

      - name: Set up Bun
        uses: oven-sh/setup-bun@0c5077e51419868618aeaa5fe8019c62421857d6 # v2.2.0
        with:
          bun-version: latest

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Format check
        run: bun run format --check

      - name: Lint
        run: bun run lint

      - name: Root audit
        run: bun audit

      - name: Test
        run: bun test

      - name: Install docs dependencies
        run: bun install --frozen-lockfile
        working-directory: docs-site

      - name: Docs audit
        run: bun audit
        working-directory: docs-site

      - name: Docs build
        run: bun run build
        working-directory: docs-site

  docker-smoke:
    runs-on: ubuntu-latest
    needs: test
    steps:
      - name: Checkout
        uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7.0.0

      - name: Build Docker image
        run: docker build -t remotecache:ci .

      - name: Start Docker container
        run: docker run -d --name remotecache-ci -e ADMIN_TOKEN=test-token -p 3000:3000 remotecache:ci

      - name: Wait for server
        run: |
          for attempt in {1..30}; do
            if curl -fsS http://127.0.0.1:3000/metrics > /tmp/remotecache-metrics.txt; then
              cat /tmp/remotecache-metrics.txt
              exit 0
            fi
            sleep 1
          done

          docker logs remotecache-ci
          exit 1

      - name: Stop Docker container
        if: always()
        run: docker rm -f remotecache-ci || true

  security:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write
    steps:
      - name: Checkout
        uses: actions/checkout@9c091bb21b7c1c1d1991bb908d89e4e9dddfe3e0 # v7.0.0

      - name: Run Trivy filesystem scan
        uses: aquasecurity/trivy-action@a9c7b0f06e461e9d4b4d1711f154ee024b8d7ab8 # v0.36.0
        with:
          scan-type: fs
          scan-ref: .
          format: sarif
          output: trivy-fs.sarif
          severity: HIGH,CRITICAL
          ignore-unfixed: true
          exit-code: '1'
          limit-severities-for-sarif: true

      - name: Upload Trivy SARIF
        if: ${{ always() && (github.event_name != 'pull_request' || github.event.pull_request.head.repo.full_name == github.repository) }}
        uses: github/codeql-action/upload-sarif@c35d1b164463ee62a100735382aaaa525c5d3496 # codeql-bundle-v2.25.6
        with:
          sarif_file: trivy-fs.sarif
```
